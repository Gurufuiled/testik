# Промт: современные микровзаимодействия и skeleton states

Ниже готовый промт, который можно использовать для Codex / Cursor по нашему проекту.

---

Ты работаешь в проекте мобильного мессенджера на React Native / Expo.  
Нужно улучшить UX интерфейса за счет современных микровзаимодействий и skeleton loading states.

Контекст:

- это не редизайн с нуля, а аккуратная полировка уже существующего интерфейса;
- нужно сохранить текущую архитектуру и существующий visual language;
- нельзя ломать текущую логику чатов, reply, voice, pin, forward, gallery sheet и login flow;
- все изменения должны выглядеть современно, плавно и уместно, без перегруженности и "дешевых" анимаций.

Что нужно сделать:

1. Добавить современные микровзаимодействия в ключевые части интерфейса.
2. Добавить skeleton states на экраны и блоки, где сейчас загрузка выглядит пусто или резко.
3. Сохранить производительность на мобильных устройствах.
4. Использовать аккуратные, быстрые, ненавязчивые анимации.

Приоритет по микровзаимодействиям:

- нажатия кнопок и action-элементов;
- появление и скрытие модалок / bottom sheet / popup menu;
- reply swipe и другие жесты в чате;
- появление новых сообщений;
- переходы состояний loading / ready / error;
- hover-like / press-like feedback для карточек, строк и quick actions;
- login screen и entry transitions.

Приоритет по skeleton states:

- список чатов;
- экран чата при начальной загрузке;
- pinned / header-related placeholders, если уместно;
- gallery / media blocks, если там есть заметная загрузка;
- login / initial app readiness states, если есть пустые паузы.

Требования к качеству:

- не добавляй хаотичные анимации ради анимаций;
- не делай интерфейс визуально тяжелым;
- избегай резких скачков layout;
- skeleton должен быть похож на будущий контент по форме;
- анимации должны улучшать ощущение качества, а не отвлекать;
- по возможности используй существующие зависимости и паттерны проекта;
- если в проекте уже есть `react-native-gesture-handler`, `Animated` или похожие инструменты, опирайся на них;
- не вноси радикальные визуальные изменения без необходимости.

Что важно в ответе:

- не просто опиши идеи, а реально внеси изменения в код;
- перед изменениями изучи текущие экраны и компоненты;
- выбери самые заметные и полезные точки улучшения;
- после изменений кратко перечисли, какие микровзаимодействия и skeleton states были добавлены;
- если какие-то места пока лучше не трогать из-за риска, явно напиши это.

Если придется выбирать, отдавай приоритет:

1. chat screen
2. chats list
3. attachment / bottom sheet UX
4. login / auth screen

Цель:

Сделать интерфейс мессенджера визуально более современным, живым и polished, как у актуальных consumer messenger apps, но без потери читаемости, производительности и уже работающей логики.

---

## Короткая версия промта

Сделай в нашем мобильном мессенджере современные микровзаимодействия и skeleton loading states без редизайна с нуля. Сохрани текущий стиль и архитектуру, не ломай существующий chat flow. Приоритет: экран чата, список чатов, attachment sheet, login screen. Анимации должны быть плавными, быстрыми и ненавязчивыми, а skeleton states должны быть похожи на реальный контент и улучшать UX загрузки. Реально внеси изменения в код, а не только опиши идеи.
